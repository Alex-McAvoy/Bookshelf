# Bookshelf
托管于个人博客中的电子书架，个人阅读记录

git 提交规范

commit message = [subject] message

常见的 subject 种类以及含义如下：

    Feat: 提交新功能
    Fix: 修复 bug
    Docs: 提交仅文档相关的修改
    Style: 提交仅格式化、标点符号、空白等不影响代码运行的代码风格变更（不影响运行逻辑）
    Refactor: 提交代码重构（既不是新增功能也不是修复bug的代码更改）
    Perf: 提交提升性能的代码修改
    Test: 提交测试相关的内容
    Chore: 杂项（构建过程、辅助工具、依赖库等相关内容）
    Build: 提交影响构建系统、外部依赖性的变更
    CI: 提交配置文件、脚本
    Revert: 回滚
    Disuse：移除废弃代码
